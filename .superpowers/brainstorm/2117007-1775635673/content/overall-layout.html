Overall Site Structure

How should the public landing page and map page relate to each other?

A

Separate Pages — Full Landing + Map App

A rich public homepage (hero, stats, about, CTA) that links to a dedicated full-screen map application at /map. The map page has its own top bar with filters and controls.

Site structure
🌍 GasWatch  |  Home  |  About  |  Open Map →
Hero: "Monitoring Global Methane Emissions"
Stats · Gas types · CTA button
About section
Data sources
Latest detections
↓ /map — Full-screen MapLibre + filter sidebar
B

Map-First — Minimal Header + Collapsible Info Panel

The map is the primary experience from the start. A slim top bar holds branding and nav. A collapsible side panel shows "About / Stats". Filters live in a floating panel over the map.

Site structure
🌍 GasWatch  |  [About ▼]  |  [Filters ▼]
← Collapsible panel
Stats / About / Legend
Full-screen map
Floating filter toolbar
C

Hybrid — Landing Scrolls Into Map

A short landing section (hero + key stats) at the top of the page, with the map embedded below the fold. Scrolling down reveals the map. Filters appear as a top bar when the map is in view.

Site structure
🌍 GasWatch  |  Home  |  About
Hero + stats (short section)
↓ Scroll → Full-screen embedded map
Sticky filter bar appears on scroll
