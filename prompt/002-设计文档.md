# GasWatch 前端设计文档

## 1. 项目概述

基于 Angular 21 的温室气体排放监测可视化平台，展示卫星遥感探测到的气体羽流数据（CH4、CO、NO2 等）。

---

## 2. 技术栈

| 层 | 技术 |
|---|---|
| 前端框架 | Angular 21 (standalone, zoneless) |
| 样式 | Tailwind CSS v4 |
| 地图 | MapLibre GL JS |
| TIFF 解析 | GeoTIFF.js |
| 测试 | Vitest |
| 后端（未来）| Go + SQLite |

---

## 3. 路由结构

```
/        → LandingComponent   (lazy-loaded)
/map     → MapComponent       (lazy-loaded)
/map?id=<plume_id>  → MapComponent，自动选中指定羽流
```

## 6. 服务层架构（Approach B）

```
src/app/core/services/
  plume.service.ts          ← abstract class (interface contract)
  mock-plume.service.ts     ← reads assets/mock/plumes.json
  api-plume.service.ts      ← calls Go REST API
```

```typescript
// plume.service.ts
export abstract class PlumeService {
  abstract getPlumes(filter: FilterCriteria): Observable<Plume[]>;
  abstract getPlumeById(id: string): Observable<Plume>;
  abstract getStats(): Observable<Stats>;
}
```

```typescript
// environment.ts
export const environment = {
  production: false,
  useMock: true,
  apiUrl: 'http://localhost:8080'
};

// app.config.ts
{
  provide: PlumeService,
  useClass: environment.useMock ? MockPlumeService : ApiPlumeService
}
```

切换到真实 API：将 `useMock` 改为 `false`，组件零改动。

## 8. Landing Page

**Section 顺序**（从上到下）:

1. **Nav bar** — sticky，logo + "Open Map →" CTA，`bg-slate-900/80 backdrop-blur`
2. **Hero** — `h-screen`，深色背景 + CSS 粒子动画，大标题，"Open Map" 按钮
3. **Stats Bar** — 4 个动态计数器（总探测数、覆盖国家、气体类型、最新探测日期），`IntersectionObserver` 触发 count-up 动画，数据来自 `PlumeService.getStats()`
4. **Gas Types** — CH4 / CO / NO2 三张卡片，各含图标、名称、简介
5. **Latest Detections** — 最新 10 条羽流卡片网格，每张含气体类型徽章、通量速率、国家、卫星、日期，"View on Map →" 链接跳转至 `/map?id=<id>`
6. **About** — 项目使命简介，探测机构署名
7. **Footer** — 品牌名、数据来源、年份

## 9. Map Page

### 9.1 布局

- 全屏 MapLibre 地图（`position: absolute; inset: 0`）
- 顶部 slim nav bar（品牌 + "← Home"）
- **左上悬浮 Filter Panel**（默认折叠为 pill，点击展开）
- **右侧悬浮 Detail Drawer**（点击 marker 后滑入）

### 9.2 地图可视化

**数据源**: GeoJSON FeatureCollection，由 `PlumeService.getPlumes(filter)` 返回后转换。

**Cluster**（MapLibre 内置 cluster，proximity-based）:
- 圆形，大小 ∝ `point_count`（小 ≤5，中 ≤20，大 20+）
- 颜色 = dominant gas type（CH4: `#fb923c`，CO: `#f87171`，NO2: `#818cf8`）
- 顶部 symbol layer 显示 count badge
- 点击 → `map.flyTo()` 居中 + zoom in → cluster 自动分裂

**Individual Marker**（孤立点）:
- 两层同心圆：外层低透明度光晕 + 内层实心点
- 颜色 = gas type
- 点击 → 触发 Plume Overlay + 打开 Detail Drawer

**Plume Overlay**（点击 marker 后渲染）:
- `line` layer：虚线多边形轮廓（来自 GeoJSON geometry）
- TIFF 解析：`GeoTIFF.js` 读取像素值 → 渲染为 MapLibre canvas 热力图
- Hover → tooltip 显示该像素点的浓度值
- 点击其他区域 / 关闭 drawer → 移除 overlay layers

### 9.3 Filter Panel

- 默认：折叠 pill，显示 active filter 数量徽章
- 展开后包含：
  - 气体类型（多选 checkbox）
  - 卫星（下拉）
  - 日期范围（date inputs）
  - 排放量区间（range slider，单位 kg/h）
- "Apply" 按钮触发 `getPlumes()` → 地图数据刷新

### 9.4 Detail Drawer

- CSS `transform: translateX()` 滑入动画
- 顶部：羽流多边形缩略图（mini canvas）
- 展示所有 `Plume` 字段
- 关闭按钮 + 点击地图其他区域关闭

## 11. Go 后端（未来）

**API 端点**:
```
GET /api/plumes                  # 支持 gasType, dateFrom, dateTo, fluxMin, fluxMax, satellite, sector
GET /api/plumes/:id
GET /api/plumes/:id/tiff         # 返回 TIFF 文件
GET /api/stats
```

**SQLite `plumes` 表**:
```sql
CREATE TABLE plumes (
  id TEXT PRIMARY KEY,
  raw_json TEXT NOT NULL,        -- 完整 _META.json GeoJSON Feature
  geometry TEXT NOT NULL,        -- GeoJSON Polygon
  tiff_path TEXT NOT NULL,       -- 相对路径，如 data/rawdata/A0000092/A0000092_PLUME.tif
  gas_type TEXT,
  satellite TEXT,
  overpass_time TEXT,
  flux_rate REAL,
  country TEXT,
  sector TEXT
);
CREATE INDEX idx_gas_type ON plumes(gas_type);
CREATE INDEX idx_overpass_time ON plumes(overpass_time);
CREATE INDEX idx_flux_rate ON plumes(flux_rate);
```

## 12. 错误处理

| 场景 | 处理方式 |
|---|---|
| `getPlumes()` 失败 | Toast 通知 + 重试按钮 |
| `getPlumeById()` 失败 | Drawer 内显示错误状态 |
| TIFF 加载失败 | 仅显示多边形轮廓，不显示热力图 |
| 网络超时 | RxJS `timeout` + `catchError` |

**Loading states**:
- 地图：获取数据时显示 skeleton overlay
- Detail Drawer：加载时显示 spinner
- Landing Stats：占位数字 → 数据到达后 count-up 动画
